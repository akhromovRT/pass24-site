# Архитектура

## Обзор

Публичный сайт pass24.online на платформе WordPress. Выполняет маркетинговую функцию: представляет продукт, генерирует лиды, содержит документацию и контакты.

## Контекст

- **Домен:** pass24.online
- **Платформа:** WordPress (self-hosted)
- **Аудитория:** потенциальные B2B-клиенты, HR/безопасники компаний, интеграторы СКУД

## Ключевые компоненты

| Компонент | Назначение |
|---|---|
| WordPress Core | CMS-движок |
| Активная тема | Внешний вид и шаблоны страниц |
| Дочерняя тема (child theme) | Кастомизации поверх основной темы |
| Плагины | Функциональные расширения (формы, SEO, кэш и др.) |
| wp-config.php | Настройки подключения к БД и константы |
| MySQL / MariaDB | База данных (контент, настройки, пользователи) |
| Медиабиблиотека | Изображения, документы, видео |

## Потоки данных

```
Посетитель → nginx/Apache → PHP (WordPress) → MySQL
                                    ↓
                             Тема + Плагины
                                    ↓
                           HTML-страница → Посетитель
```

Формы захвата лидов → (email-уведомление / CRM-интеграция)

## Технологии и зависимости

- **CMS:** WordPress (актуальная стабильная версия)
- **Язык:** PHP 8.x
- **БД:** MySQL 8 / MariaDB 10.x
- **Веб-сервер:** Apache или nginx
- **Деплой:** FTP / SSH + WP-CLI / плагин для деплоя
- **SEO:** Yoast SEO или Rank Math
- **Формы:** Contact Form 7 / WPForms
- **Кэширование:** WP Super Cache / W3 Total Cache
- **Безопасность:** Wordfence или iThemes Security

## Нефункциональные требования и ограничения

- Сайт должен загружаться быстро (Core Web Vitals)
- Адаптивный дизайн (mobile-first)
- Кастомный код только в дочерней теме — не редактировать файлы основной темы напрямую
- Резервное копирование перед любыми изменениями на продакшне
- Пароли и ключи API — только в `.env` или wp-config.php, не в коде
