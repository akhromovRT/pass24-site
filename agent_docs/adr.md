### ADR-001: Bricks Builder вместо Elementor

**Дата:** 2026-02-23
**Статус:** Принято

**Контекст:**
Нужен конструктор страниц WordPress с максимальной производительностью. Два кандидата: Bricks Builder и Elementor Pro.

**Решение:**
Выбран Bricks Builder ($79, пожизненная лицензия).

**Обоснование:**
- Bricks генерирует семантически чистый HTML без лишних оберток
- Лучшие показатели Core Web Vitals из всех WP-конструкторов
- Критично для KPI: LCP < 2.5 сек и PageSpeed ≥ 90
- Elementor добавляет ~0.18 сек к времени загрузки (задокументировано в benchmark)
- Одноразовая оплата vs ежегодная подписка Elementor ($59–$999/год)

**Компромиссы:**
- Меньше готовых шаблонов, чем у Elementor
- Меньше экосистема сторонних аддонов

### ADR-002: Gravity Forms + Flamix вместо WPForms

**Дата:** 2026-02-23
**Статус:** Принято

**Контекст:**
Нужна форма с нативной интеграцией Битрикс24, условной логикой, многошаговостью и автозахватом UTM.

**Решение:**
Gravity Forms ($59/год) + плагин Flamix Bitrix24.

**Обоснование:**
- Flamix — нативный коннектор к Битрикс24 с автозахватом UTM
- Gravity Forms поддерживает частичные отправки (partial entries) — лид создаётся после Шага 1
- WPForms Pro также поддерживает Flamix, но Gravity Forms более гибкий для сложной логики
- Нет необходимости в кастомном webhook-коде

**Компромиссы:**
- Платный плагин ($59/год)
- Требует дополнительный плагин Flamix

### ADR-003: Rank Math Pro вместо Yoast SEO

**Статус:** Принято

**Контекст:**
Нужен SEO-плагин с поддержкой Schema, AI-функциями и интеграцией с Google Search Console.

**Решение:**

- Лучшая поддержка Schema markup из коробки (SoftwareApplication, FAQ, LocalBusiness)

### ADR-004: n8n для автопубликации контента

**Дата:** 2026-02-23
**Статус:** Принято

**Контекст:**
Нужна автоматизация публикации SEO-контента через Claude API в WordPress.

**Решение:**
n8n (self-hosted или облако) → Claude API → WordPress REST API.

**Обоснование:**
- n8n поддерживает все нужные шаги: cron → API → WordPress REST → уведомление в Битрикс24
- Визуальный редактор воркфлоу без кода
- Нативная интеграция с Bitrix24, Pexels, WordPress, OpenAI/Anthropic
- Self-hosted вариант: нет ограничений на количество выполнений
- Все статьи публикуются как черновики — человек проверяет перед публикацией

**Компромиссы:**
- Требует отдельный сервер для self-hosted (или подписка на n8n Cloud)
- Нужно настроить Application Password в WordPress

### ADR-005: Архитектура лид-скоринга через n8n + GA4

**Дата:** 2026-02-23
**Статус:** Принято

**Контекст:**
Нужна модель скоринга лидов для автоматической маршрутизации в Битрикс24.

**Решение:**
Поле `UF_CRM_LEAD_SCORE` в Битрикс24 обновляется через n8n-воркфлоу на основе событий GA4 + целей Яндекс.Метрики.

**Обоснование:**
- Поведенческие сигналы (посещение тарифов, использование калькулятора) важнее демографических
- n8n связывает GA4 Measurement Protocol с Битрикс24 REST API без кастомного кода
- Порог 70 баллов для передачи в сделку основан на конверсионной аналитике аналогичных B2B-продуктов

### ADR-006: Разделение документации на маркетинговый blueprint и техническое ТЗ

**Дата:** 2026-02-25
**Статус:** Принято

**Контекст:**
Единый `blueprint.md` (1300+ строк) содержал и маркетинговые тексты, и техническую спецификацию. Документ был слишком объёмным для эффективной работы, а контент-менеджер и разработчик читали одно и то же, хотя им нужны разные части.

**Решение:**
Разделить на два документа:
- `agent_docs/blueprint.md` — маркетинговый blueprint (позиционирование, контент, CTA, SEO)
- `agent_docs/technical-spec.md` — техническое ТЗ (стек, плагины, Битрикс24, аналитика, API)

**Обоснование:**
- Каждый документ адресован своей аудитории (маркетолог / разработчик)
- Меньший объём = быстрее находить нужное и меньше контекста для AI-агентов
- Можно обновлять тексты без риска сломать техническую конфигурацию и наоборот

### ADR-007: Прозрачные тарифы на сайте

**Дата:** 2026-02-25
**Статус:** Принято

**Контекст:**
Текущий сайт (pass24online.ru на Creatium) скрывает все 4 тарифа — ни одна цена не показана. Посетитель должен связаться с менеджером, чтобы узнать стоимость. Исследование SaaS-лидеров показало, что прозрачность цен повышает конверсию на 20–30% для B2B SaaS.

**Решение:**
Публиковать реальные цены на странице `/pricing/`:
- Lite: 10 990 ₽/мес
- Standard: 14 990 ₽/мес
- Prof: 24 990 ₽/мес
- Business: 48 990 ₽/мес
- Скидка 10% при оплате за год
- Пометка «Безлимитные пользователи и пропуска»

**Обоснование:**
- Скрытые цены создают барьер для квалифицированных лидов — они уходят к конкурентам, которые показывают стоимость
- Прозрачность повышает доверие (Столп 7: масштаб и доверие)
- Лиды, которые видели цену и всё равно оставили заявку, более квалифицированные

**Компромиссы:**
- Конкуренты видят цены (но могут узнать их и без сайта)
- Нужно оперативно обновлять при изменении прайса

### ADR-008: Единственный Hero + 2 варианта для A/B вместо 6

**Дата:** 2026-02-25
**Статус:** Принято

**Контекст:**
Старый blueprint предлагал 6 вариантов Hero-заголовков для A/B-тестирования. Анализ показал, что все 6 были слабыми: общие формулировки без конкретики и цифр, «облачная СКУД» как главный месседж (непонятно аудитории УК).

**Решение:**
Один сильный основной вариант + 2 для A/B:
- **Основной:** «Пропускной режим, который работает сам» + бейдж «Новатор Москвы» + цифры (300+, 1.5 млн)
- **B:** «Пока охрана записывает номера — ваши жители стоят в очереди» (от проблемы)
- **C:** «1,5 миллиона пропусков в месяц. Ни одного бумажного.» (от масштаба)

**Обоснование:**
- Лидеры мировых SaaS используют формулу: заголовок 4–6 слов + социальное доказательство выше линии сгиба
- 3 варианта дают статистически значимый результат быстрее, чем 6 (при минимум 500 посетителей на вариант)
- Каждый вариант тестирует свою гипотезу: решение / боль / масштаб

### ADR-009: 8 вертикальных решений вместо 6

**Дата:** 2026-02-25
**Статус:** Принято

**Контекст:**
Старый blueprint покрывал 6 сегментов (ЖК, КП, БЦ, Склады, Стройка, ЧОП). Анализ верифицированного списка 87 клиентов показал значительное присутствие PASS24 в сегментах, не отражённых на сайте.

**Решение:**
Расширить до 8 вертикалей:
1. ЖК (было) — Садовые Кварталы, Full House, Розмарин
2. КП (было) — Агаларов Эстейт, Росинка, Николино
3. БЦ (было) — Нагатино i-Land (МТС), ТехноПарк Цвет
4. **Логистика и склады** (новое) — MLP, ФК Пульс, Сходня-Инжиниринг
5. **Производство** (новое) — МетМебель, Флексо-Пак, НПО Промпат
6. **Индустриальные парки** (новое) — ЭЛМА Всеволжск, ИП Орёл, Предпортовый
7. Стройка (было)
8. ЧОП (было)

**Обоснование:**
- MLP (1,7 млн м²) и ФК Пульс (14 ЛЦ) — крупнейшие клиенты, но их сегмент не представлен
- Индустриальные парки — растущий сегмент с 3+ клиентами
- Каждая новая вертикаль приносит отдельный SEO-кластер (кластер 8)
- Увеличивает охват long-tail запросов

### ADR-010: Политика «только верифицированные факты»

**Дата:** 2026-02-25
**Статус:** Принято

**Контекст:**
В старом blueprint обнаружены ложные данные:
- Логотипы Villagio Estate, ПИК, Самолет, PSN Group, CEO ROOMS — не являются клиентами
- Упоминание «Группа ЛАНИТ» — убрано по указанию клиента
- Награды EMERGE 2023, PROESTATE 2022 — нет подтверждающих дипломов
- «1,5 млн заявок в год» — устаревшая цифра (правильно: в месяц)

**Решение:**
Ввести строгую политику верификации:
- Каждый факт на сайте должен иметь источник (прайс-лист, диплом, подтверждение клиента)
- Список разрешённых клиентов для социального доказательства — только из верифицированной базы 87 объектов
- Запрещённый список: Villagio, ПИК, Самолет, PSN Group, CEO ROOMS, «Группа ЛАНИТ», EMERGE 2023, PROESTATE 2022

**Обоснование:**
- Ложные социальные доказательства подрывают доверие при проверке
- Верифицированные клиенты (МТС, MLP, Садовые Кварталы) сильнее неподтверждённых
- Премия «Новатор Москвы» (Правительство Москвы, Собянин) ранее отсутствовала в blueprint, хотя является самой значимой наградой

**Компромиссы:**
- Меньше логотипов для полосы доверия (7 вместо 12+)
- Нужно запрашивать согласие клиентов на использование логотипов

### ADR-011: Пилот 14 дней как центральный оффер

**Дата:** 2026-02-25
**Статус:** Принято

**Контекст:**
Текущий сайт использует CTA «Узнать подробнее» и «Запросить демо» без конкретики. Конкуренты требуют предоплату до демонстрации. PASS24 реально предоставляет 14-дневный бесплатный пилот с обучением и онлайн-сопровождением.

**Решение:**
Сделать «Бесплатный пилот 14 дней» единым центральным оффером на всех страницах:
- Hero: «Попробовать бесплатно — 14 дней»
- Sticky-бар: «Попробовать бесплатно — 14 дней»
- Финальный CTA: «Запустить пилот — 14 дней бесплатно»
- Под каждым CTA: «✓ Развёртывание за 1 час ✓ Обучение включено ✓ Без замены оборудования»

**Обоснование:**
- Конкретный срок («14 дней») работает лучше абстрактного «бесплатная демонстрация» — снижает барьер входа
- Включение обучения и сопровождения снимает страх «не разберёмся»
- «Без замены оборудования» закрывает главный страх УК — дорогой переход
- Формула проверена SaaS-лидерами: бесплатный триал с ограничением по времени конвертирует в 2x лучше, чем freemium
