# Архивация и временные файлы

## Архивация старых файлов

Чек-лист при замене важных файлов (README, конфиги, документация):
- Убедиться, что есть `archive/` (создать при отсутствии).
- Имя: `[YYYY-MM-DD_HH-MM]_original_name.ext`.
- В начало файла добавить префикс:

  ```markdown
  <!--
  АРХИВИРОВАНО: 2025-01-13 20:15
  ПРИЧИНА: Краткая причина
  НОВАЯ ВЕРСИЯ: ../path/to/new/file
  -->
  ```
- Переместить файл в `archive/`.
- Не архивировать: временные файлы, git-файлы, секреты, большие бинарники, файлы из `.gitignore`.

## Временные файлы
- Все временные артефакты складывать в `temp/` в корне репозитория.
- По окончании задачи очищать `temp/`.
- Добавлять новые временные файлы в `.gitignore` при необходимости.
